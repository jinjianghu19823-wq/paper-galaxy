# 隐私

[English](PRIVACY.md) | 简体中文

Paper Galaxy 默认本地优先。英文版仍是规范版本；本文件提供简体中文说明。

- 用户文档默认留在本地。
- 没有遥测。
- 没有自动上传。
- 不要求账号。
- 应用应该可以离线使用。
- Phase 2 默认把抽取文本和文本块存储在 `.paper-galaxy/paper_galaxy.sqlite3`。
- Phase 4 在 SQLite 中存储紧凑的抽取报告，并可写出可选的本地 JSON sidecar。这些报告不包含完整抽取文本。
- Phase 5 可以把可选的 dense 文档/文本块向量存储在本地 SQLite 中。只有用户运行 `paper-galaxy embed` 时才会生成向量。
- Phase 6 的手动主题簇标签覆盖只存储在本地 SQLite 中。
- Phase 6 的 pair explanations 会在本地 CLI 或浏览器应用中展示共享词项和短文本块摘录。这些摘录可能仍包含敏感源文本，但 Paper Galaxy 不会上传它们。
- Phase 7 把保存的地图运行元数据、点坐标、主题簇标签、top terms 和最近邻摘要存储在本地 SQLite 中。保存的运行不存储完整源文档或文本块文本。
- Phase 7 验证报告包含计数、schema 状态、警告和错误，不包含完整抽取文本。
- Phase 7 备份导出只有在用户用 `--yes` 确认后才包含本地 SQLite 数据库。默认不包含源文档。
- Phase 7 插件元数据是静态且本地的；没有远程插件加载或扩展市场。
- `missing` 和 `unindexed` 记录可能保留之前抽取的本地文本和文本块，以便本地索引可以恢复文档历史。
- Phase 3 默认启动绑定到 `127.0.0.1` 的本地服务器。
- 浏览器应用只与本地后端通信。
- Phase 3 静态资源由本地服务，不引用 CDN、远程字体或外部图片。
- 浏览器应用不会上传文档、收集遥测或调用远程服务。
- Phase 3.1 只把图谱 UI 偏好和手动固定节点位置存储在浏览器 `localStorage` 中，key 由本地数据库身份和文档 ID 组成。这些布局值不会上传，也不会写入 SQLite。
- 可选 OCR 只在本地运行。它可能需要用户安装 Tesseract 等本地 OCR 二进制，但 Paper Galaxy 不会把图片、抽取文本、OCR 输出或抽取报告上传到远程服务。
- 可选 embeddings 只在本地运行。默认拒绝远程 Sentence Transformer 模型名以避免隐藏下载；使用 `--allow-model-download` 是用户对模型解析/下载的显式选择。
- 主题簇标签和 pair explanations 都从本地索引文本生成；没有强制 LLM 或远程标签服务。
- 选择非 loopback host 可能会把应用暴露给局域网中的其他设备。
- `.paper-galaxy/` 是本地项目状态，并已被 gitignore。
- 删除 `.paper-galaxy/` 会删除该项目的本地 Paper Galaxy 数据库和项目元数据。

未来如果增加云功能，必须是 opt-in。它们应清楚解释哪些数据会离开本机、会去哪里，以及用户如何关闭或删除这些数据。

## 公开演示

GitHub Pages 演示是静态的，只使用 `examples/tiny_corpus` 的合成内容。它不连接后端，不上传文档，不加载远程运行时资源，也不包含本地 SQLite 数据库。演示 JSON 包含元数据、图谱点、标签、词项、邻居和短示例解释摘录，不包含完整源文档文本。

公开站点包含英文和简体中文页面。两者使用同一组合成演示数据。

## 未来云库说明

个人云库文档描述的是未来可能的 opt-in 设计。它们并没有在当前代码库中实现云同步、账号、托管索引或上传行为。

如果未来实现云功能，备份、向量、地图运行、标签和元数据都必须被视为敏感数据。用户必须能在启用前理解哪些数据会离开设备。
